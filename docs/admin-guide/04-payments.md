# Section 4: Payments

The **Payments** section provides a centralised ledger of all transactions recorded across your institution's payment forms. To access it, click **Payments** in the left navigation sidebar.

![Figure 24: The All Transactions page](/payforms-docs/assets/images/Picture24.png)

At the top of the page, four summary cards display the following:

- **Today's Volume** — The total amount collected today across all payment forms
- **Outstanding Balance** — The cumulative value of payments that are pending or partially paid
- **Success Rate** — The ratio of fully paid and partially paid transactions to total payment attempts
- **Today's Transactions** — The total number of payment attempts recorded today

Three action buttons are available in the top-right corner of the page:

- **Refresh** — Reloads the transaction list to reflect the latest payment activity
- **Export CSV** — Downloads the current transaction list as a CSV file for offline reporting
- **Offline Payments** — Shortcut to record a payment made outside the platform

---

## Transaction Table

Below the summary cards, all transactions are listed in a table with the following columns:

- **Transaction ID** — A unique reference code assigned to each payment attempt
- **Date** — The date the transaction was initiated
- **Contact** — The name of the payer, if available in your Contacts records
- **Form** — The payment form the transaction was submitted through
- **Amount** — The value of the transaction in Naira (₦)
- **Method** — How the payment was made (e.g. Pay Online)
- **Status** — The current state of the transaction: **Paid**, **Pending**, **Partial**, or **Failed**
- **Actions** — Options to **Edit** or **View** the transaction record

You can search for a specific transaction using the **Search by Reference or Contact** bar, filter by status using the **All Statuses** dropdown, or narrow results by date using **Filter by Date**.

---

## Viewing a Transaction

To view the full details of a transaction, click **View** in the **Actions** column of the relevant record.

![Figure 26: The Transaction Details screen](/payforms-docs/assets/images/Picture26.png)

The **Transaction Details** screen displays the following:

- **Status** — The current payment status (e.g. Failed, Paid)
- **Amount** — The transaction value in Naira (₦)
- **Date** — The date the transaction was recorded
- **Transaction ID** — The system-generated internal identifier for the transaction
- **Reference** — The unique reference code used to track the transaction across the platform

Two buttons are available at the bottom of the screen:

- **View History** — Displays a timestamped log of all status changes for this transaction
- **View Form** — Opens the payment form through which the transaction was submitted
